# 个人博客系统项目需求文档

## 一、项目背景与目标

### 1.1 项目背景
随着校园生活的结束，未来的工作节奏将更加紧凑，缺乏专门的时间来整理和沉淀学习过程中积累的技术知识。当前面临的问题包括：
- 学习内容杂乱，存在“学了就忘”、“记不住细节”的情况；
- 笔记资料频繁丢失，因系统重装或硬盘损坏等原因；
- 缺乏统一的知识管理平台。

为解决上述问题，计划开发一个**个人博客系统**，用于：
- **记录学习笔记、总结心得**；
- **展示技术能力与作品**；
- **集中存储个人资源，防止数据丢失**；
- **通过实践提升自身技术水平**。

### 1.2 项目目标
构建一个功能完善、易于维护、安全性良好的个人博客系统，满足个人知识管理和对外展示的需求，同时作为技术成长过程中的实战项目。

## 二、功能需求分析

### 2.1 核心功能模块

| 功能模块   | 功能描述                                                     | 优先级 |
| ---------- | ------------------------------------------------------------ | ------ |
| 文章管理   | 支持文章的发布、展示、编辑、删除，支持 Markdown 格式及所见即所得编辑器 | 必须   |
| 分类与标签 | 支持分类与标签管理，便于归档与检索                           | 必须   |
| 评论系统   | 支持用户评论                                                 | 必须   |
| 搜索功能   | 支持关键词、分类、标签搜索                                   | 必须   |
| 访问统计   | 提供后台访问量统计、来源分析                                 | 次要   |

### 2.2 扩展功能模块

| 功能模块      | 功能描述                                             |
| ------------- | ---------------------------------------------------- |
| 资源上传      | 支持上传并展示小说、软件、漫画等文件资源             |
| Live2D 看板娘 | 集成可交互的 Live2D 模型，提供类似豆包风格的互动体验 |
| 邮箱登录/验证 | 支持邮箱注册、登录及验证功能（待定）                 |
| 黑暗模式      | 支持页面主题切换，提供黑暗模式与亮色模式             |
| 站内留言      | 提供访客留言入口，收集建议与反馈                     |

### 2.3 用户相关功能

#### ① 多用户系统
- 支持用户通过邮箱注册、登录、找回密码；
- 注册时发送邮箱验证码，验证成功后方可完成注册；
- 登录后用户可进行评论、资源上传、下载等操作；
- 预留用户权限扩展接口，便于后期接入会员、管理员角色。

#### ② 积分系统
- 所有注册用户拥有独立积分账户；
- **获取方式**：
  - 每日签到奖励（例如 +0.5 积分/天）；
  - 系统赠送（注册奖励、活动奖励等）；
- **使用方式**：
  - 下载特定资源需消耗积分；
  - 后续可扩展充值、兑换、开通会员等功能。

#### ③ 文件资源下载控制
- 上传资源时可设置：
  - 免费下载；
  - 需消耗积分下载；
  - 会员专属资源（预留）；
- 所有下载操作需记录日志，防止盗链和刷积分行为。

#### ④ 邮箱验证码系统
- 在以下场景中发送验证码：
  - 用户注册；
  - 密码找回；
  - 邮箱变更；
- 验证码具备有效期，过期自动失效，确保安全。

### 2.4 安全策略

| 安全措施 | 实现方式                                                     |
| -------- | ------------------------------------------------------------ |
| 密码加密 | 使用加盐哈希算法（如 bcrypt、PBKDF2）加密存储用户密码        |
| 接口鉴权 | 所有涉及用户身份的操作均需进行身份认证与权限校验             |
| 权限控制 | 所有接口实施权限检查，防止未授权访问                         |
| 防刷机制 | 对高频访问接口（如注册、登录、邮件发送）实施频次限制或验证码防护 |

## 三、技术选型方案

| 技术类别         | 技术方案                                   | 备注                             |
| ---------------- | ------------------------------------------ | -------------------------------- |
| 前端框架         | Vue.js（含 Vue Router、Pinia 等）          | 个人熟悉，适合快速开发           |
| 后端框架         | Spring Boot + MyBatis                      | Java 生态成熟，适合教学与实战    |
| 数据库           | MySQL                                      | 常用关系型数据库，便于维护与扩展 |

| 权限框架（选做） | 自研简单权限模块 / Shiro / Spring Security | 后期可按需引入，优先自研简版     |
| 日志系统         | Logback / SLF4J                            | 用于系统运行日志记录             |
| 构建工具         | Maven / Gradle                             | 依赖管理与项目构建               |

## 四、UI设计与交互规划

### 4.1 整体风格
- **极简主义**：界面简洁明了，避免视觉干扰；
- **清爽舒适**：配色柔和，提升阅读与操作体验；
- **响应式布局**：适配 PC 与移动端访问。

### 4.2 主色调
- **浅灰白底色**：避免纯白色刺眼，符合现代 UI 设计趋势；
- 辅助色采用蓝/绿系，突出重点内容与交互按钮。

### 4.3 界面目标
- **视觉舒适**：整体色彩协调、层次分明；
- **功能清晰**：导航明确，操作路径直观；
- **易于使用**：减少用户认知负担，提升交互效率。
